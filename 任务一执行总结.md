### 2. 事件数据结构 ✅

```typescript
interface EventState {
  [key: string]: any;
}

interface EventLog {
  eventId: string; // 事件唯一 ID (UUID v4)
  eventType: EventType; // 事件类型
  accountId?: number; // 账户 ID
  orderId?: number; // 订单 ID
  positionId?: number; // 持仓 ID
  version: number; // 版本号
  beforeState: EventState; // 事件前的状态
  afterState: EventState; // 事件后的状态
  timestamp: Date; // 时间戳
  metadata?: Record<string, any>; // 元数据
}
```

### 3. 事件存储原则 ✅

**禁止仅保存最终结果**:
- 必须记录完整的 beforeState 和 afterState
- beforeState 记录事件前的状态
- afterState 记录事件后的状态
- 禁止只保存 afterState

**必须记录的关键信息**:
- eventId: UUID v4，事件唯一标识
- accountId: 账户 ID（相关时）
- orderId: 订单 ID（相关时）
- positionId: 持仓 ID（相关时）
- timestamp: 事件发生时间
- version: 事件版本号
- metadata: 额外元数据

**状态记录原则**:
- 状态 = 事件回放结果
- beforeState 和 afterState 可以用于状态回放
- 支持基于事件的历史重建

## 技术实现

### 1. 事件记录流程

```
执行关键操作
    ↓
获取 beforeState
    ↓
执行操作
    ↓
获取 afterState
    ↓
┌─────────────────────────────┐
│  1. 获取当前版本号              │
│     - 从 event_log 获取当前版本号         │
│     - 版本号自动递增                  │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│  2. 记录事件                    │
│     - 生成 eventId (UUID v4)            │
│     - 插入 beforeState           │
│     - 插入 afterState            │
│     - 记录 timestamp            │
│     - 记录 version               │
│     - 存储到 event_log 表        │
└─────────────────────────────┘
    ↓
返回 eventId
```

### 2. 事件回放流程

```
获取 eventId 和 targetVersion
    ↓
┌─────────────────────────────┐
│  1. 查询事件的所有版本          │
│     - 按版本号升序排列              │
│  2. 获取目标版本的状态              │
│  - fromState = event_log.after_state │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│  3. 逐版本应用状态                  │
│     - 获取每个版本的 afterState       │
│     - 验证状态一致性                 │
│     - 应用到系统                    │
│     - 验证结果                      │
│     - 如果成功，记录回放成功             │
│     - 如果失败，记录回放错误             │
└─────────────────────────────┘
    ↓
更新 replay_status
    ↓
SUCCESS / FAILED
```
