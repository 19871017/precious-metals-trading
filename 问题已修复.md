# 问题修复说明

## 问题：500 错误 - 数据库缓存导致

### 错误信息
```
GET http://localhost:3001/shuhai/quote?code=CENQA0 500 (Internal Server Error)
获取实时行情失败: Error: 获取实时行情失败: 500
```

### 根本原因

后端在查询数据库缓存时，如果数据库表不存在或连接失败，会抛出异常导致整个请求失败。

### 解决方案

已修改 `server/src/routes/shuhai.ts`，采用**优雅降级**策略：

1. **数据库查询失败时自动跳过** - 不影响API正常响应
2. **异步保存到数据库** - 不阻塞响应，即使数据库不可用也能正常工作
3. **内存缓存作为后备** - 确保始终有缓存层可用

### 缓存流程（修复后）

```
前端请求
    ↓
后端API
    ↓
检查数据库缓存 (5秒/60秒) [try-catch保护]
    ↓ 失败 → 跳过
检查内存缓存
    ↓ 未命中
数海API
    ↓
返回数据 + 存入内存 + 异步存入数据库[不阻塞]
```

### 使用说明

#### 无需数据库也能正常工作！

系统会自动检测数据库状态：
- ✅ **数据库可用** → 使用数据库缓存（持久化）
- ✅ **数据库不可用** → 使用内存缓存（重启后丢失）
- ✅ **两者都失败** → 直接请求API

#### 启动服务即可

```bash
# 启动后端
cd server
npm run dev

# 启动前端
cd ..
npm run dev
```

#### 可选：初始化数据库

如果需要持久化缓存，可以初始化数据库：

```bash
cd server

# 1. 配置环境变量
copy .env.example .env
# 编辑 .env 设置数据库密码

# 2. 创建数据库
psql -U postgres
CREATE DATABASE precious_metals_trading;
\q

# 3. 初始化表
npm run db:init-shuhai
```

### 技术细节

#### 修改的代码

**实时行情接口 (`/shuhai/quote`)**:
```typescript
// 数据库查询加入 try-catch
try {
  const cachedData = await getQuoteFromDB(code);
  if (cachedData) { ... }
} catch (dbError) {
  console.log(`[数据库] 数据库查询失败，跳过数据库缓存`);
}

// 异步保存，不阻塞响应
saveQuoteToDB(code, shuhaiCode, formatted).catch(err => {
  console.log(`[数据库] 保存行情缓存失败: ${err.message}`);
});
```

**K线接口 (`/shuhai/kline`)**:
- 同样的处理方式

#### 错误日志

如果数据库不可用，会看到如下日志（不影响功能）：
```
[数据库] 数据库查询失败，跳过数据库缓存: ...
[数据库] 保存行情缓存失败: ...
```
