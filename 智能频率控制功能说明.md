# 智能频率控制功能说明

## 功能概述

实现了智能API调用频率控制，能够根据数海API的限流响应自动调整数据更新频率，在保证实时性的同时避免触发API限流。

## 工作原理

### 核心参数

| 参数 | 值 | 说明 |
|------|---|------|
| `MIN_INTERVAL` | 1000ms (1秒) | 最小更新间隔，防止请求过于频繁 |
| `MAX_INTERVAL` | 10000ms (10秒) | 最大更新间隔，确保数据不会过于陈旧 |
| `RATE_LIMIT_COOLDOWN` | 60000ms (60秒) | 限流冷却时间，避免过于敏感的频率调整 |
| `currentUpdateInterval` | 1000ms (初始值) | 当前实际使用的更新间隔 |

### 频率调整策略

#### 1. 增加间隔（降低频率）

触发条件：
- 检测到API限流错误（407或429状态码）
- 在冷却时间内（60秒）再次触发限流

调整规则：
- 当前间隔 × 2
- 最大不超过 `MAX_INTERVAL` (10秒)
- 立即重启定时器应用新间隔

示例：
```
1000ms → 2000ms → 4000ms → 8000ms → 10000ms (最大值)
```

#### 2. 降低间隔（提高频率）

触发条件：
- 成功获取到数据
- 距离上次限流超过冷却时间（60秒）
- 当前间隔大于 `MIN_INTERVAL`

调整规则：
- 当前间隔 × 0.8
- 最小不低于 `MIN_INTERVAL` (1秒)
- 立即重启定时器应用新间隔

示例：
```
10000ms → 8000ms → 6400ms → 5120ms → 4096ms → 3276ms → 2620ms → 2096ms → 1677ms → 1000ms (最小值)
```

## 使用场景

### 场景1：正常情况
```
初始状态: 1000ms
持续更新: 1000ms (1秒更新一次)
数据实时性: 高
```

### 场景2：触发限流
```
第1次限流: 1000ms → 2000ms (间隔翻倍)
第2次限流: 2000ms → 4000ms (再次限流，继续翻倍)
第3次限流: 4000ms → 8000ms
第4次限流: 8000ms → 10000ms (达到最大值)
```

### 场景3：恢复频率
```
在10000ms间隔下稳定运行60秒
检测到数据更新成功: 10000ms → 8000ms
下次成功: 8000ms → 6400ms
...
逐渐降低到: 1000ms
```

## 优势

1. **自动适应**：无需手动调整，系统根据API响应自动优化
2. **平衡性能**：在实时性和稳定性之间找到最佳平衡点
3. **避免崩溃**：防止因频繁限流导致数据完全不可用
4. **快速恢复**：一旦API恢复正常，自动降低间隔提高实时性
5. **保护价格缓存**：配合价格缓存机制，即使限流也能显示有效价格

## 配置建议

根据不同的使用场景，可以调整以下参数：

### 场景A：对实时性要求极高
```typescript
MIN_INTERVAL = 500;        // 0.5秒
MAX_INTERVAL = 5000;        // 5秒
RATE_LIMIT_COOLDOWN = 30000; // 30秒
```

### 场景B：平衡型（当前配置，推荐）
```typescript
MIN_INTERVAL = 1000;        // 1秒
MAX_INTERVAL = 10000;       // 10秒
RATE_LIMIT_COOLDOWN = 60000; // 60秒
```

### 场景C：稳定性优先
```typescript
MIN_INTERVAL = 2000;        // 2秒
MAX_INTERVAL = 30000;       // 30秒
RATE_LIMIT_COOLDOWN = 120000; // 2分钟
```

## 注意事项

1. **冷却时间**：不要设置过短，避免频繁调整导致不稳定
2. **最大间隔**：不要设置过大，确保数据不会过于陈旧
3. **最小间隔**：不要设置过小，超过数海API的300ms限制
4. **价格缓存**：配合前端的价格缓存机制，即使限流也能显示有效价格

## 未来优化方向

1. **分品种控制**：不同品种使用不同的更新频率
2. **时间段控制**：交易时间和非交易时间使用不同频率
3. **动态冷却**：根据限流频率动态调整冷却时间
4. **预测性调整**：基于历史数据预测可能的限流时间
5. **优先级控制**：重要品种更新频率更高

## 总结

智能频率控制通过自动感知API限流情况并动态调整更新间隔，在保证数据实时性的同时有效避免API限流，为用户提供稳定可靠的行情数据服务。
