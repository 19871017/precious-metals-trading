## 快速开始

### 第一步：配置环境变量

在 `server` 目录下创建 `.env` 文件：

```bash
cd server
copy .env.example .env
```

然后编辑 `.env` 文件，设置数据库密码：

```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=precious_metals_trading
DB_USER=postgres
DB_PASSWORD=你的数据库密码

SHUHAI_USERNAME=wu123
SHUHAI_PASSWORD=wu123
```

### 第二步：确保PostgreSQL运行

检查PostgreSQL服务是否运行：

**Windows:**
```bash
sc query postgresql-x64-16
```

**Linux/Mac:**
```bash
pg_isready
```

如果服务未运行，启动PostgreSQL服务。

### 第三步：创建数据库（如果不存在）

连接到PostgreSQL：

```bash
psql -U postgres
```

创建数据库：

```sql
CREATE DATABASE precious_metals_trading;
\q
```

### 第四步：初始化缓存表

运行初始化脚本：

```bash
cd server
npm run db:init-shuhai
```

或者使用tsx直接运行：

```bash
npx tsx src/scripts/init-shuhai-cache.ts
```

成功后会看到：
```
✅ 数海API缓存表初始化完成!
创建的表: [ 'shuhai_quote_cache', 'shuhai_kline_cache' ]
```

### 第五步：启动后端服务

```bash
cd server
npm run dev
```

### 第六步：启动前端服务

```bash
npm run dev
```

## 缓存架构说明

### 三层缓存结构

```
前端请求
    ↓
后端API
    ↓
检查数据库缓存 (5秒/60秒)
    ↓ 未命中
检查内存缓存
    ↓ 未命中
数海API
    ↓
返回数据 + 存入内存 + 存入数据库
```

### 缓存时间

- **实时行情**: 5秒
- **K线数据**: 60秒
- **数据库持久化**: 数据不丢失，重启后可用

## 常见问题

### Q1: 数据库连接失败

**错误信息**: `ECONNREFUSED 127.0.0.1:5432`

**解决方法**:
1. 检查PostgreSQL服务是否运行
2. 检查 `.env` 中的数据库配置
3. 确认端口是否正确（默认5432）

### Q2: K线图不显示

**可能原因**:
1. 数据库表未初始化
2. 数海API返回格式错误

**解决方法**:
1. 运行初始化脚本：`npm run db:init-shuhai`
2. 查看浏览器控制台错误
3. 查看后端日志：`[数海代理] K线数据获取成功`

### Q3: 表已存在错误

**解决方法**:
- 先删除表再初始化：
```sql
DROP TABLE IF EXISTS shuhai_kline_cache;
DROP TABLE IF EXISTS shuhai_quote_cache;
\q
```
- 然后重新运行初始化脚本

## 数据维护

### 查看缓存命中率

后端日志会显示缓存来源：
- `[数据库] 从数据库获取行情缓存: GOLD`
- `[内存] 从内存缓存获取K线: GOLD, 周期: 60`
- `[数海代理] 从API获取实时行情: GOLD -> CMGCA0`

### 清理过期数据

```sql
-- 删除30天前的K线数据
DELETE FROM shuhai_kline_cache 
WHERE cached_at < NOW() - INTERVAL '30 days';

-- 删除旧行情缓存
DELETE FROM shuhai_quote_cache 
WHERE cached_at < NOW() - INTERVAL '1 hour';
```

## 性能优势

1. **减少API调用**: 大部分请求命中缓存
2. **降低响应时间**: 数据库读取比API快
3. **提高稳定性**: API故障时缓存仍可用
4. **持久化存储**: 重启后缓存数据不丢失
