## 核心功能

### 1. 并发下单模拟 ✅

**配置参数**:
- userCount: 并发用户数（默认 30）
- orderCountPerUser: 每用户下单数（默认 10）
- 总订单数: 30 * 10 = 300 单

**实现特点**:
- 30 个用户同时下单
- 每个用户下单 10 次
- 并发执行，模拟真实场景
- 支持重试机制（最多 3 次）
- 记录每个订单的延迟

### 2. 行情连续跳变 ✅

**配置参数**:
- marketSymbols: 行情品种列表（默认 6 个品种）
- priceChangeRange: 价格变化范围（默认 ±10%）
- volatilityInterval: 跳变间隔（默认 500ms）

**实现特点**:
- 6 个品种同时跳变
- 每 500ms 跳变一次
- 价格变化范围：±10%
- 持续 5 分钟

### 3. 批量强平触发 ✅

**触发方式**:
- 通过降低账户余额触发强平
- 通过行情跳变触发强平
- 批量测试强平机制

**实现特点**:
- 模拟用户亏损
- 降低账户余额到强平线
- 触发自动强平
- 测试强平调度器

### 4. 测试报告输出 ✅

**最大延迟**:
- 记录所有订单的处理延迟
- 统计最大延迟时间
- 统计最小延迟时间

**失败率**:
- 统计总订单数
- 统计成功订单数
- 统计失败订单数
- 计算成功率

**资金一致性检查**:
- 初始总余额
- 最终总余额
- 差异检查（允许 ±0.01 误差）
- 用户级别一致性详情

## 使用示例

### 1. 运行默认混沌测试

```bash
curl -X POST http://localhost:3001/system/chaos-test/run
```

### 2. 运行自定义混沌测试

```bash
curl -X POST http://localhost:3001/system/chaos-test/run \
  -H "Content-Type: application/json" \
  -d '{
    "userCount": 50,
    "orderCountPerUser": 20,
    "durationMinutes": 10,
    "marketSymbols": ["XAUUSD", "CMGCA0"],
    "priceChangeRange": 20
  }'
```

### 3. 获取测试结果

```bash
# 获取测试状态
curl http://localhost:3001/system/chaos-test/status

# 获取测试结果
curl http://localhost:3001/system/chaos-test/results

# 获取测试报告
curl http://localhost:3001/system/chaos-test/report
```

## 注意事项

1. **测试用户**: 测试会创建大量测试用户，使用前请确认
2. **数据清理**: 测试后建议清理测试数据
3. **系统影响**: 混沌测试会对系统产生较大负载
4. **测试环境**: 建议在测试环境运行
5. **资金影响**: 强平会真实扣除用户资金，请谨慎操作
6. **并发控制**: 并发用户数和订单数应根据系统能力调整
7. **行情模拟**: 行情模拟仅为压力测试，不影响真实行情
