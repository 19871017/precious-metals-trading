# 后台管理系统访问说明

## 系统分离架构

前端系统和后台管理系统已经完全分离，使用独立的入口和认证系统。

## 访问方式

### 前端系统
- **URL**: `http://localhost:5173`
- **用途**: 用户交易系统
- **认证**: 使用普通用户账号登录

### 后台管理系统
- **URL**: `http://localhost:5173/admin-entry.html`
- **用途**: 管理员后台管理系统
- **认证**: 使用独立的管理员账号登录

## 默认管理员账号

### 测试账号
- **用户名**: `admin`
- **密码**: `admin123`

### 备用测试账号
- **用户名**: `superadmin`
- **密码**: `super123`

## 访问路径说明

### 1. 直接访问后台管理系统
```
http://localhost:5173/admin-entry.html
```

### 2. 从前端系统进入
1. 登录前端系统
2. 进入"我的"页面
3. 点击"后台管理"卡片
4. 系统会自动跳转到 `/admin-entry.html` 登录页面

### 3. 后台管理系统内部路由
- **登录页**: `/admin-entry.html`
- **仪表盘**: `/admin-entry.html#/dashboard`

## 认证系统说明

### 前端认证
- Token存储: `localStorage.getItem('token')`
- 用户信息: `localStorage.getItem('user')`

### 后台管理认证（独立）
- Token存储: `localStorage.getItem('adminToken')`
- 用户信息: `localStorage.getItem('adminUser')`

两个系统的认证完全独立，互不影响。

## 退出登录

### 前端系统
- 清除 `token` 和 `user`
- 返回登录页

### 后台管理系统
- 清除 `adminToken` 和 `adminUser`
- 返回后台登录页 `/admin/login`

## 功能模块

后台管理系统包含以下功能模块：
1. **仪表盘** - 系统概览和统计数据
2. **用户管理** - 用户账号管理
3. **代理管理** - 代理层级和佣金管理
4. **产品管理** - 交易品种配置
5. **订单管理** - 订单查询和管理
6. **持仓管理** - 用户持仓监控
7. **财务管理** - 充值提现审核
8. **分佣管理** - 佣金结算管理
9. **风控管理** - 风险控制和强平管理
10. **系统设置** - 手续费和系统参数配置

## 安全提示

1. 仅授权管理员可以访问后台管理系统
2. 后台管理系统使用独立的认证系统
3. 请勿在公共设备上保存密码
4. 定期修改管理员密码
5. 生产环境建议使用更复杂的密码

## 技术架构

```
前端系统 (App.tsx)
  ↓
用户登录认证
  ↓
前端功能页面

后台管理系统 (AppAdmin.tsx)
  ↓
管理员登录认证 (AdminProtectedRoute)
  ↓
后台管理功能 (AdminPC.tsx)
```

两个系统共享同一 React 代码库，但使用不同的入口文件和路由配置。

## 注意事项

1. 开发环境需要运行 `npm run dev`
2. 生产环境构建会生成两个入口：`index.html` 和 `admin.html`
3. 修改了 `vite.config.ts` 配置多入口构建
4. 新增了 `AppAdmin.tsx` 作为后台管理系统入口
5. 新增了 `AdminProtectedRoute.tsx` 作为后台认证保护组件
