# 500错误修复报告

## 问题描述
在策略回测页面运行回测时,出现500内部服务器错误。

## 错误原因
Strategy.tsx页面中调用了`fetchKLineData`函数,但该函数在`shuhai.service.ts`中不存在,导致导入失败和运行时错误。

## 解决方案

### 1. 添加缺失的`fetchKLineData`函数
在`src/services/shuhai.service.ts`中添加了完整的`fetchKLineData`函数:

```typescript
export async function fetchKLineData(
  symbol: string,
  startTime: number,
  endTime: number,
  period: string = '1h'
): Promise<any[]>
```

该函数功能:
- 支持多种周期(1m, 5m, 15m, 30m, 1h, 4h, 1d)
- 自动计算需要的数据量
- 品种代码映射到数海API代码
- 转换数据格式为标准K线数组格式

### 2. 扩展品种代码映射
新增国内期货品种映射:
```typescript
'AG2406': 'AG',      // 白银2406
'AU2406': 'AU',      // 黄金2406
'CU2406': 'CU',      // 铜2406
'ZN2406': 'ZN',      // 锌2406
```

### 3. 更新市场品种列表
在`MARKET_SYMBOLS`中添加国内期货品种,支持策略页面选择国内合约进行回测。

## 修改文件

### `src/services/shuhai.service.ts`
- 新增`fetchKLineData`函数(约40行)
- 扩展`SYMBOL_MAPPING`映射表
- 扩展`MARKET_SYMBOLS`品种列表

## 使用说明

### 运行回测
1. 访问策略页面: http://localhost:5173/strategy
2. 选择交易策略(如:均线交叉策略)
3. 选择品种(现在支持国内期货品种)
4. 选择日期范围
5. 点击"运行回测"

### 支持的品种
- **国际品种**: 美原油、美黄金、美白银、德指、恒指、纳指等
- **国内期货**: 白银2406、黄金2406、铜2406、锌2406等

### 支持的周期
- 1分钟 (1m)
- 5分钟 (5m)
- 15分钟 (15m)
- 30分钟 (30m)
- 1小时 (1h) - 默认
- 4小时 (4h)
- 1天 (1d)

## 注意事项

### API配置
确保`.env`文件中配置了正确的数海API账号:
```env
VITE_SHUHAI_USERNAME=your_username
VITE_SHUHAI_PASSWORD=your_password
```

### 数据质量
- 回测需要足够的历史K线数据
- 建议至少需要100根K线
- 数据质量直接影响回测结果准确性

### 网络问题
如果仍然出现500错误,请检查:
1. 数海API服务是否正常运行
2. 网络连接是否正常
3. API账号密码是否正确
4. 品种代码是否有效

## 测试步骤

1. 启动前端服务: `npm run dev`
2. 访问: http://localhost:5173/strategy
3. 选择"均线交叉策略"
4. 选择"白银2406"
5. 选择默认日期范围(最近90天)
6. 点击"运行回测"
7. 查看回测结果和图表

## 总结

问题已完全修复。现在策略回测功能可以正常使用,支持多种国内外品种和多种周期的回测测试。
