## 核心功能

### 1. 独立 Worker Pool

✅ **Worker 数量 = CPU核心数 / 2**
- 自动计算 Worker 数量
- 支持手动配置
- 动态调整

✅ **独立队列**
- 队列名称: `risk-engine:queue`
- 与 HTTP 请求线程、撮合线程完全隔离
- 使用 BullMQ 实现

✅ **高并发处理**
- 每个 Worker 并发处理 5 个任务
- 总并发能力 = (CPU核心数 / 2) * 5

### 2. 任务优先级

✅ **支持 0-10 的优先级**
- 0-2: 紧急风控检查
- 3-5: 正常风控检查
- 6-8: 批量风控检查
- 9-10: 后台风控分析

### 3. 风险保障

✅ **行情波动时仍能运行**
- 独立资源池，不受其他线程影响
- 高优先级任务优先处理
- 失败自动重试
- 队列持久化，服务重启不丢失

## 技术实现

### 架构设计

```
Express HTTP 服务器 (HTTP 请求线程)
           ↕
      (不共享资源)
           ↕
SystemPriorityController (撮合线程)
           ↕
      (不共享资源)
           ↕
Risk Engine Worker Pool (CPU核心数 / 2 个 Worker)
  独立队列: risk-engine:queue
  每个 Worker 并发: 5
```

### Worker Pool 配置

```typescript
{
  workerCount: Math.ceil(os.cpus().length / 2), // CPU核心数 / 2
  concurrency: 5, // 每个 Worker 并发处理 5 个任务
  queueName: 'risk-engine:queue', // 独立队列
  maxRetries: 3, // 最大重试次数
  retryDelay: 1000, // 重试延迟 (ms)
}
```

### 使用示例

```typescript
// 方式 1: 同步等待结果
const result = await riskEngine.validateAsync(request, 5, 5000);

// 方式 2: 异步提交 + 等待结果
const { jobId } = await riskEngine.validateWithWorker(request, 5);
const result = await riskEngine.waitForValidationResult(jobId, 5000);

// 方式 3: 直接使用（不走 Worker Pool）
const result = await riskEngine.validate(request);
```
