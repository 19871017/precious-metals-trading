#### 实现的风控检查

✅ **账户权益实时计算**
- 实时计算账户权益 = 余额 + 冻结金额 + 浮动盈亏
- 获取用户所有持仓的未实现盈亏
- 支持高频实时更新

✅ **保证金占用比例**
- 计算保证金使用率 = 占用保证金 / 账户权益
- 实时监控保证金使用情况
- 根据保证金使用率确定风险等级

✅ **杠杆上限校验**
- 默认最大杠杆: 100x
- 可通过配置动态调整
- 检查用户请求的杠杆是否超过上限

✅ **单用户最大持仓限制**
- 默认最大持仓数: 10 个
- 实时统计用户当前持仓数
- 超过限制则拒绝新开仓

✅ **单品种最大风险敞口**
- 默认最大敞口: 1000 手
- 统计单个品种的总持仓量
- 防止单品种风险过度集中

✅ **总平台风险敞口统计**
- 默认最大敞口: 10,000,000
- 统计整个平台的保证金占用
- 防止平台整体风险过高

✅ **高频交易限流**
- 默认每秒最大下单次数: 5 次
- 使用 Redis 实现分布式限流
- 防止用户高频刷单

✅ **冷却时间机制**
- 默认冷却时间: 1000 ms
- 防止用户连续快速下单
- 有效防止刷单攻击
