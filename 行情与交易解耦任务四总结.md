#### 实现的功能

✅ **Redis 行情缓存**
- 使用 Redis 存储实时行情数据
- 缓存键: `market:cache:{productCode}`
- 缓存TTL: 30 秒
- 支持批量更新

✅ **行情快照**
- 行情快照存储: `market:snapshot:{productCode}`

- 支持批量创建

✅ **版本号机制**
- 当前版本号存储: `market:version:current`
- 行情更新时自动增加版本号
- 版本号用于校验行情有效性

✅ **版本号校验**
- 验证行情缓存版本是否一致
- 版本过低则拒绝使用
- 避免"旧行情成交"

✅ **"旧行情成交"检测**
- 检测行情缓存是否过期(默认5秒)
- 过期的行情会被拒绝使用
- 防止使用过期数据成交

### 1. 行情缓存流程

```
行情更新 (MarketDataServiceV2)
    ↓
1. 更新行情到数据库
    ↓
2. 更新 Redis 缓存
    ↓
3. 创建行情快照
    ↓
4. 增加全局版本号
    ↓
完成
```

### 2. 交易撮合流程

```
交易请求 (TradingService)
    ↓
1. 从 Redis 获取行情缓存
    ↓
2. 验证行情版本号
    ↓
3. 检测"旧行情成交"
    ↓
4. 使用缓存行情执行撮合
    ↓
5. 完成交易
```

### 3. 版本号校验流程

```
订单执行前
    ↓
获取行情缓存
    ↓
获取缓存版本号
    ↓
获取当前全局版本号
    ↓
比较版本号
    ↓
缓存版本 >= 当前版本?
    ↓
  ┌─ 是 ────────→ 继续执行
  └─ 否 ────────→ 拒绝订单
```

### 4. "旧行情成交"防护流程

```
订单执行前
    ↓
获取行情缓存
    ↓
检查缓存时间
    ↓
当前时间 - 缓存时间 > maxAgeMs?
    ↓
  ┌─ 是 ────────→ 拒绝订单
  └─ 否 ───────→→ 继续执行
```
