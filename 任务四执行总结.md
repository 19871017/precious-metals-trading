## 核心功能

### 1. 独立的优先级队列 ✅

- 队列名称: `liquidation:priority`
- 与普通强平队列 `orders:liquidation` 分离
- 使用 BullMQ 优先级队列
- 支持高并发处理（并发数: 10）
- 更高的限流配置（100 次/秒）

### 2. 保证金率最低账户优先 ✅

**优先级计算逻辑**:

- **CRITICAL 账户**（marginUsage >= 1 或 equity <= 0）
  - 优先级范围: 0-50
  - 计算公式: `Math.floor(marginUsage * 50)`
  - 保证金率越低，优先级越高

- **DANGER 账户**（0.8 <= marginUsage < 1.0）
  - 优先级范围: 51-100
  - 计算公式: `51 + Math.floor(marginUsage * 49)`
  - 保证金率越低，优先级越高

**示例**:
```
CRITICAL 账户:
- marginUsage = 1.0 → priority = 50
- marginUsage = 0.95 → priority = 47
- marginUsage = 0.9 → priority = 45
- marginUsage = 0.85 → priority = 42

DANGER 账户:
- marginUsage = 0.99 → priority = 100
- marginUsage = 0.9 → priority = 96
- marginUsage = 0.85 → priority = 93
- marginUsage = 0.8 → priority = 90
```

### 3. 禁止 FIFO 强平 ✅

- 所有需要强平的仓位同时加入队列
- 不按持仓创建时间顺序
- 按保证金率优先级处理
- BullMQ 按优先级处理任务（0 最高）

### 4. 高速处理 ✅

- Worker 并发数: 10
- 限流: 100 次/秒
- 重试次数: 3 次
- 重试延迟: 500ms（指数退避）

```
│         LiquidationSchedulerV2                       │

                      ↕
                 (独立队列)
                      ↕
┌─────────────────────────────────────────────────────┐
│        LiquidationPriorityScheduler                 │
│        (优先级队列 - 非FIFO)                         │
│              队列: liquidation:priority              │
│       保证金率最低账户优先强平                         │
└─────────────────────────────────────────────────────┘
```

### 扫描流程

```
启动优先级强平调度器
    ↓
每 500ms 执行一次扫描循环
    ↓
┌─────────────────────────────────────┐
│  1. 扫描风险账户                   │
│     - 获取所有有持仓的账户          │
│     - 计算账户权益和保证金率         │
│     - 确定风险等级                 │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  2. 按保证金率排序                 │
│     - 保证金率最低排在最前          │
│     - CRITICAL 优先于 DANGER        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  3. 计算优先级                     │
│     - CRITICAL: 0-50               │
│     - DANGER: 51-100               │
│     - 保证金率越低，优先级越高      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  4. 同时加入所有需要强平的仓位      │
│     - 禁止 FIFO                    │
│     - 每个仓位设置相应的优先级       │
└─────────────────────────────────────┘
    ↓
BullMQ 按优先级处理任务（0 最高）
```

- `GET /liquidation/priority/queue/stats` - 获取优先级队列统计
