## 📌 重要说明

**关于数据库选择：**

## 3. 数据库技术栈

### 3.1 PostgreSQL

**版本要求：** PostgreSQL 14+ (推荐 15 或 16)

**用途：**
- 存储用户数据
- 存储订单、持仓、交易记录
- 存储财务数据
- 存储代理和佣金数据
- 存储系统配置

**关键特性使用：**
- **JSONB** - 灵活存储配置数据
- **数组类型** - 存储权限列表
- **事务支持** - 确保数据一致性
- **索引优化** - 提升查询性能
- **窗口函数** - 复杂查询支持
- **分区表** - 大数据量支持（可扩展）

**为什么不使用 MySQL？**
1. JSONB 功能不如 PostgreSQL 完善
2. 窗口函数支持较晚
3. 复杂类型支持有限
4. 并发控制机制不如 PostgreSQL

### 3.2 Redis

**版本要求：** Redis 7.0+ (最低 6.0)

**用途：**
- 用户会话存储
- 实时行情数据缓存
- API 请求限流
- 热数据缓存
- 发布订阅（实时消息）

**数据结构使用：**
- **String** - 缓存键值对
- **Hash** - 存储行情数据
- **List** - 消息队列
- **Set** - 去重和集合操作

## 9. 关键技术决策

### 9.1 为什么选择 TypeScript？

**优势：**
- 提供类型安全，减少运行时错误
- 改善代码可维护性
- 更好的 IDE 支持
- 提前发现潜在问题

**应用范围：**
- ✅ 前端：所有 React 组件、工具函数
- ✅ 后端：所有 API 路由、服务、工具

### 9.2 为什么选择 PostgreSQL 而非 MySQL？

**PostgreSQL 优势：**
- **JSONB** - 更灵活的文档存储
- **窗口函数** - 复杂查询支持
- **并发控制** - MVCC 机制更优
- **数据类型** - 丰富的原生类型
- **扩展性** - 支持插件和自定义函数

**金融场景优势：**
- 高精度数值计算支持
- 强事务一致性
- 复杂查询能力

### 9.3 为什么选择 React 而非 Vue？

**选择理由：**
- 团队熟悉程度
- 生态系统更成熟
- 企业级应用经验丰富
- 第三方库支持更广

### 9.4 为什么选择 TDesign？

**选择理由：**
- 腾讯开源，质量保证
- 企业级设计语言
- 组件丰富且稳定
- 支持多种设计主题
