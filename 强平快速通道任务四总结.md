- DANGER 账户优先级: 51-100

- 所有需要强平的仓位同时加入队列
- 不按持仓创建时间顺序
- 按保证金率优先级处理

✅ **高速处理**
- Worker 并发数: 10

## 技术实现

### 1. 扫描风险账户

```typescript
private async scanRiskAccounts(): Promise<RiskAccountPriority[]> {
  // 获取所有有持仓的账户
  const accounts = await query(`SELECT * FROM accounts WHERE balance > 0 AND frozen_amount > 0`);

  for (const account of accounts) {
    // 获取账户的所有持仓
    const positions = await query(`SELECT * FROM positions WHERE user_id = $1 AND status = 1`, [userId]);

    // 计算未实现盈亏
    let totalUnrealizedPnl = 0;
    for (const position of positions) {
      const currentPrice = await this.getMarketData(position.productCode);
      const unrealizedPnl = this.calculateUnrealizedPnl(position, currentPrice);
      totalUnrealizedPnl += unrealizedPnl;
    }

    // 计算保证金使用率
    const equity = account.balance + totalUnrealizedPnl;
    const marginUsage = equity > 0 ? account.frozen_amount / equity : 1;

    // 确定风险等级
    let riskLevel: 'CRITICAL' | 'DANGER';
    if (equity <= 0 || marginUsage >= 1) {
      riskLevel = 'CRITICAL';
    } else if (marginUsage >= 0.8) {
      riskLevel = 'DANGER';
    }
  }
}
```

### 2. 按保证金率排序

```typescript
private sortAccountsByMarginUsage(accounts: RiskAccountPriority[]): RiskAccountPriority[] {
  // 按保证金率降序排序（保证金率越低，优先级越高）
  return accounts.sort((a, b) => b.marginUsage - a.marginUsage);
}
```

### 3. 计算优先级并加入队列

```typescript
private async queuePriorityLiquidations(accounts: RiskAccountPriority[]): Promise<number> {
  for (const account of accounts) {
    // 计算优先级
    let basePriority: number;
    if (account.riskLevel === 'CRITICAL') {
      // CRITICAL: 0-50 (保证金率越低，优先级越高)
      basePriority = Math.floor(account.marginUsage * 50);
    } else {
      // DANGER: 51-100
      basePriority = 51 + Math.floor(account.marginUsage * 49);
    }

    // 同时加入所有需要强平的仓位（禁止 FIFO）
    for (const position of account.positions) {
      if (!position.shouldLiquidate) {
        continue;
      }

      await this.addPriorityLiquidationJob({
        userId: account.userId,
        positionId: position.positionId,
        productCode: position.productCode,
        liquidationPrice: position.liquidationPrice,
        reason: '保证金率过低触发强平（快速通道）',
        marginUsage: account.marginUsage,
        priority: basePriority, // 设置优先级
      });
    }
  }
}
```

### 4. 处理优先级强平任务

```typescript
private async processPriorityLiquidation(job: Job<PriorityLiquidationJobData>): Promise<void> {
  const { userId, positionId, liquidationPrice, reason, priority } = job.data;

  // 获取分布式锁
  const lock = await acquirePositionLock(positionId, userId);
  if (!lock) {
    logger.warn(`获取持仓锁失败,跳过: positionId=${positionId}`);
    return;
  }

  try {
    // 执行强平
    await this.executePriorityLiquidationInternal(
      userId,
      positionId,
      liquidationPrice,
      reason
    );

    logger.info(
      `优先级强平执行成功: userId=${userId}, positionId=${positionId}, priority=${priority}`
    );
  } finally {
    // 释放锁
    await releaseLock(lock);
  }
}
```

## 注意事项

1. **优先级队列**: 优先级队列与普通强平队列独立运行，互不影响
2. **优先级设置**: 保证金率越低，优先级越高（数值越小）
3. **禁止 FIFO**: 所有需要强平的仓位同时加入队列，不按创建时间顺序
4. **分布式锁**: 使用分布式锁防止同一仓位被同时强平
5. **事务处理**: 强平在数据库事务中完成，确保数据一致性
6. **审计日志**: 完整记录强平操作，便于审计和排查

## 使用场景

### 场景 1: 行情剧烈波动

当行情剧烈波动时，多个账户同时触发强平：

1. 优先级强平调度器扫描所有风险账户
2. 按保证金率排序（最低优先）
3. 计算优先级（CRITICAL: 0-50, DANGER: 51-100）
4. 所有需要强平的仓位同时加入队列
5. BullMQ 按优先级处理（保证金率最低的最先处理）
