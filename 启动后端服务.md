# 启动后端服务

## 方法1：双击运行（推荐）

双击运行以下文件：
```
server\start-server.bat
```

## 方法2：命令行启动

打开新的终端窗口，执行：

```bash
cd c:\Users\WY\Desktop\precious-metals-trading\server
npm run dev
```

## 方法3：使用PowerShell

```powershell
cd c:\Users\WY\Desktop\precious-metals-trading\server
npm run dev
```

---

## 预期输出

启动成功后会看到类似以下内容：

```
> precious-metals-trading@1.0.0 dev
> tsx watch src/index.ts

[数海代理] 清理过期缓存: quote(0)->0, batch(0)->0
服务器运行在 http://localhost:3001
```

## 验证服务

在浏览器或使用curl访问：

```bash
# 健康检查
curl http://localhost:3001/shuhai/health

# 或在浏览器打开
http://localhost:3001/shuhai/health
```

应该返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "status": "ok",
    "service": "shuhai-proxy",
    ...
  }
}
```

## 前端访问

后端启动后，前端应该能正常获取数据：

- 前端地址：http://localhost:5173
- 后端API：http://localhost:3001

## 如果遇到问题

### 问题1：模块找不到

确保在 `server` 目录下执行命令：
```bash
cd server
npm run dev
```

### 问题2：端口被占用

如果3001端口被占用，可以修改：
- 编辑 `server/src/index.ts`
- 修改端口号

### 问题3：依赖缺失

```bash
cd server
npm install
```

## 常见错误处理

### 数据库错误（正常）

如果看到以下日志，不影响功能：
```
[数据库] 数据库查询失败，跳过数据库缓存: ...
```

系统会自动使用内存缓存。

### 数海API错误

如果数海API返回407错误，系统会自动重试单个请求。
