## 核心功能

### 1. Watchdog 服务 ✅

实现了完整的 Watchdog 服务，包含以下核心功能：

**定时检测**:
- 检测间隔: 2 秒（可配置）
- 自动启动和停止
- 持续监控系统状态

**订单处理耗时检测**:
- 跟踪正在处理的订单
- 检测订单处理阶段
- 超过阈值自动记录

**数据库锁等待检测**:
- 查询 PostgreSQL 锁等待情况
- 计算平均和最大锁等待时间
- 超过阈值自动记录

**Redis 延迟检测**:
- 执行 PING 命令测试延迟
- 维护延迟历史记录（最多 100 条）
- 计算 P95、P99 延迟
- 超过阈值自动记录

**阻塞事件记录**:
- 自动记录到 `block_event_log` 表
- 计算严重程度（LOW/MEDIUM/HIGH/CRITICAL）
- 记录详细数据（JSON）

**支持自动上传到 Git**:
- 配置选项: `enableAutoUpload`
- 上传逻辑预留接口
- 可手动触发上传

### 2. 配置参数 ✅

| 参数 | 默认值 | 说明 |
|------|--------|------|
| checkInterval | 2000 | 检测间隔 (ms) |
| orderProcessingThreshold | 5000 | 订单处理超时阈值 (ms) |
| dbLockWaitThreshold | 1000 | 数据库锁等待超时阈值 (ms) |
| redisLatencyThreshold | 100 | Redis 延迟超时阈值 (ms) |
| enableAutoUpload | false | 是否自动上传到 Git |

### 3. 阻塞事件类型 ✅

| 事件类型 | 说明 |
|---------|------|
| ORDER_PROCESSING_SLOW | 订单处理慢 |
| DB_LOCK_WAIT | 数据库锁等待 |
| REDIS_LATENCY_HIGH | Redis 延迟过高 |

### 4. 严重程度分级 ✅

| 严重程度 | 条件 |
|---------|------|
| CRITICAL | value >= threshold * 2 |
| HIGH | threshold * 1.5 <= value < threshold * 2 |
| MEDIUM | threshold <= value < threshold * 1.5 |
| LOW | value < threshold |

## 技术实现

### 1. 订单处理跟踪

```typescript
// 开始跟踪
watchdogService.startTrackingOrder(orderId, userId, productCode);

// 更新阶段
watchdogService.updateOrderPhase(orderId, 'VALIDATION');
watchdogService.updateOrderPhase(orderId, 'MATCHING');
watchdogService.updateOrderPhase(orderId, 'NOTIFICATION');

// 停止跟踪
watchdogService.stopTrackingOrder(orderId);
```

### 2. 阻塞事件检测流程

```
启动 Watchdog 服务
    ↓
每 2 秒执行一次检测循环
    ↓
┌─────────────────────────────────────┐
│  1. 检测订单处理耗时               │
│     - 遍历所有活跃订单             │
│     - 检查处理时间是否超过阈值      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  2. 检测数据库锁等待               │
│     - 查询 pg_locks 和             │
│       pg_stat_activity              │
│     - 计算锁等待时间               │
│     - 检查是否超过阈值             │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  3. 检测 Redis 延迟                │
│     - 执行 redis.ping()            │
│     - 记录延迟到历史                │
│     - 计算统计信息                 │
│     - 检查是否超过阈值             │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  4. 记录阻塞事件                   │
│     - 计算严重程度                 │
│     - 插入到 block_event_log        │
│     - 如果启用了自动上传，上传到 Git │
└─────────────────────────────────────┘
```

### 3. 数据库表结构

```sql
CREATE TABLE block_event_log (
  id SERIAL PRIMARY KEY,
  event_type VARCHAR(50) NOT NULL,
  timestamp TIMESTAMP NOT NULL,
  threshold NUMERIC NOT NULL,
  actual_value NUMERIC NOT NULL,
  severity VARCHAR(20) NOT NULL,
  details JSONB,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 使用示例

### 1. 在订单创建时使用 Watchdog

```typescript
// src/routes/api.ts
import { watchdogService } from '../services/WatchdogService';

router.post('/order/create', async (req, res) => {
  const { userId, productCode, ... } = req.body;

  // 开始跟踪订单处理
  const orderId = await createOrderId();
  watchdogService.startTrackingOrder(orderId, userId, productCode);

  try {
    // 风控检查
    watchdogService.updateOrderPhase(orderId, 'VALIDATION');
    await validateOrder(...);

    // 订单撮合
    watchdogService.updateOrderPhase(orderId, 'MATCHING');
    await matchOrder(...);

    // 发送通知
    watchdogService.updateOrderPhase(orderId, 'NOTIFICATION');
    await sendNotification(...);

    res.json({ success: true, orderId });
  } catch (error) {
    res.status(500).json({ error: '订单创建失败' });
  } finally {
    // 停止跟踪
    watchdogService.stopTrackingOrder(orderId);
  }
});
```

### 2. 配置 Watchdog

```typescript
// 启用自动上传到 Git
watchdogService.updateConfig({
  enableAutoUpload: true,
});

// 调整阈值
watchdogService.updateConfig({
  orderProcessingThreshold: 3000, // 3 秒
  dbLockWaitThreshold: 500, // 0.5 秒
  redisLatencyThreshold: 50, // 50ms
});
```

### 3. 生成报告

```typescript
// 生成最近 24 小时的报告
const endDate = new Date();
const startDate = new Date(endDate.getTime() - 24 * 60 * 60 * 1000);

const report = await watchdogService.generateBlockEventReport(startDate, endDate);

console.log(report);
// {
//   summary: { total_events: 150, critical_count: 5, ... },
//   byType: [...],
//   bySeverity: [...],
//   recentEvents: [...]
// }
```
