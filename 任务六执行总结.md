## 核心功能

### 1. 系统负载监控 ✅

实现了完整的系统负载监控：

- **CPU 监控**: 从 SystemPriorityController 获取 CPU 使用率
- **订单延迟监控**: 从 Redis 获取订单延迟
- **检测间隔**: 每 5 秒检测一次（可配置）
- **自动判断**: 自动判断是否需要降级

### 2. 自动降级触发 ✅

实现了自动降级触发逻辑：

- **触发条件**:
  - CPU > 80% 或订单延迟 > 300ms 触发 DEGRADED
  - CPU > 90% 或订单延迟 > 500ms 触发 CRITICAL

- **降级级别**:
  - NORMAL: 正常运行
  - DEGRADED: 已降级
  - CRITICAL: 严重降级

### 3. 接口降级管理 ✅

实现了完整的接口降级管理：

**降级接口** (非必要接口):
- 历史记录查询:
  - /api/order/history
  - /api/position/history
  - /api/trade/history
- 资产统计刷新:
  - /api/account/balance
  - /api/account/equity
  - /api/account/assets
  - /portfolio
- 代理报表:
  - /api/agent/reports
  - /api/agent/statistics
  - /api/agent/performance

**核心接口** (不降级):
- 订单相关: /api/order/create, /api/order/close, /api/order/modify, /api/order/cancel
- 风控相关: /risk, /liquidation
- 系统监控: /health, /system/load

### 4. 自动恢复 ✅

实现了自动恢复机制：

- 系统负载恢复正常后自动恢复
- 恢复所有降级接口
- 记录降级时长
- 记录恢复事件

### 5. 降级中间件 ✅

实现了统一的接口降级检查中间件：

- 检查接口是否在降级列表中
- 检查接口当前状态
- 返回 503 状态码（Service Degraded）
- 友好的错误提示

## 技术实现

### 1. 降级触发流程

```
启动降级服务
    ↓
每 5 秒执行一次检测循环
    ↓
┌─────────────────────────────────────┐
│  1. 获取 CPU 使用率              │
│     - 从 SystemPriorityController   │
│     - 阈值: 80%                   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  2. 获取订单延迟                  │
│     - 从 Redis 获取                │
│     - 阈值: 300ms                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  3. 判断是否触发降级             │
│     - CPU > 80% 或                │
│     - 订单延迟 > 300ms            │
└─────────────────────────────────────┘
    ↓
触发降级或恢复
```

### 2. 接口降级检查

```typescript
// 中间件使用
app.use('/api/order/history', checkDegradation('/api/order/history'), handler);

// 内部实现
isInterfaceDegraded(interfacePath: string): boolean {
  const isDegraded = this.config.degradedInterfaces.includes(interfacePath);
  if (!isDegraded) {
    return false;
  }
  const status = this.interfaceStatus.get(interfacePath);
  return status === false;
}
```

### 3. 降级响应

```json
{
  "code": "SERVICE_DEGRADED",
  "message": "服务暂时降级，请稍后再试",
  "data": {
    "interface": "/api/order/history",
    "reason": "系统负载过高，该接口已暂时降级"
  },
  "timestamp": 1709140800000
}
```

## API 接口

### 降级服务管理

- `GET /system/degradation/status` - 获取降级服务状态
- `POST /system/degradation/start` - 启动降级服务
- `POST /system/degradation/stop` - 停止降级服务
- `PUT /system/degradation/config` - 更新降级配置
- `POST /system/degradation/trigger` - 手动触发降级
- `POST /system/degradation/recover` - 手动恢复
- `GET /system/degradation/interface/:path` - 检查接口是否降级
- `GET /system/degradation/history` - 获取降级事件历史

## 使用示例

### 1. 在订单路由中应用降级中间件

```typescript
// src/routes/api.ts
import { checkDegradation } from '../middleware/degradation';

// 历史记录查询接口（会降级）
router.get('/order/history', checkDegradation('/api/order/history'), async (req, res) => {
  // 如果接口已降级，会返回 503
  // 不会执行到这里
  const history = await getOrderHistory(req.user.id);
  res.json({ code: 0, data: history });
});

// 订单创建接口（不会降级）
router.post('/order/create', async (req, res) => {
  // 核心交易接口，不使用降级中间件
  const result = await createOrder(req.body);
  res.json({ code: 0, data: result });
});
```

### 2. 手动触发/恢复降级

```typescript
// 手动触发降级
await degradationService.manualTrigger('手动触发降级');

// 手动恢复
await degradationService.manualRecover();
```

### 3. 配置降级服务

```typescript
// 调整阈值
degradationService.updateConfig({
  cpuThreshold: 70, // 降低到 70%
  orderLatencyThreshold: 200, // 降低到 200ms
  checkInterval: 3000, // 提高检测频率到 3 秒
});
```
