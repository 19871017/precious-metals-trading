#### 实现的功能

✅ **每 500ms 扫描一次**
- 默认扫描间隔: 500ms
- 可配置扫描间隔 (100ms - 10000ms)
- 支持动态调整

✅ **扫描所有持仓账户保证金率**
- 获取所有账户及其持仓
- 获取实时行情数据
- 计算账户权益 = 余额 + 浮动盈亏
- 计算保证金使用率 = 占用保证金 / 账户权益

✅ **标记风险账户**
- 确定风险等级 (SAFE/WARNING/DANGER/CRITICAL)
- CRITICAL/DANGER 账户标记到 Redis
- 设置 60 秒过期时间
- 实时更新风险状态

✅ **进入强平队列**
- CRITICAL 账户的持仓检查是否触发强平
- 触发强平的持仓加入 BullMQ 队列
- 记录强平原因和价格

✅ **使用分布式锁**
- 使用 Redlock 算法实现分布式锁
- 锁命名: `lock:position:{positionId}:{userId}`
- 锁超时: 30 秒
- 重试次数: 3 次, 重试延迟: 100ms
- 使用 Lua 脚本确保原子性

✅ **原子扣减持仓**
- 使用数据库事务 (FOR UPDATE) 锁定持仓
- 检查持仓状态,防止重复强平
- 原子更新持仓状态为 CLOSED
- 原子更新账户余额

✅ **同步更新账户余额**
- 更新 balance += unrealizedPnl
- 更新 available_balance += unrealizedPnl + marginReleased
- 更新 frozen_amount -= marginReleased
- 更新 realized_pl += unrealizedPnl
- 所有更新在同一事务中完成

✅ **记录强平日志**
- 记录审计日志到 audit_logs 表
- 记录订单处理日志到 order_processing_logs 表
- 记录强平详细信息 (价格、盈亏、原因等)
- 所有日志包含时间戳
